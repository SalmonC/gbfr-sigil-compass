# GBFR Factor Planner 设计系统

> 页面专属规则可放在 `design-system/gbfr-factor-planner/pages/`；页面规则只能细化本文件，不能绕过可访问性、安全和设计令牌要求。

## 1. 视觉方向

风格定义为“克制的幻想工具界面”：借鉴 GBFR 的夜蓝、象牙白、雾金和清晰层级，不使用游戏素材，也不复刻参考站的蓝紫渐变、卡片比例或按钮排布。

禁止霓虹、玻璃拟态、赛博字体、大面积渐变、强发光、悬浮卡片墙、营销页式大标题和泛用 AI 文案。界面应像一件稳定、耐看的桌面工具，而不是网页展示页。

## 2. 颜色令牌

| Token | Value | 用途 |
|---|---|---|
| `--color-canvas` | `#F2EFE8` | 页面底色 |
| `--color-surface` | `#FBF9F4` | 主面板 |
| `--color-surface-muted` | `#E8E4DA` | 次级面板、禁用底色 |
| `--color-ink` | `#1C2833` | 主文本、夜蓝 |
| `--color-ink-muted` | `#5C6872` | 辅助文本 |
| `--color-line` | `#C9C3B7` | 默认边框 |
| `--color-accent` | `#A98545` | 雾金强调、激活边框 |
| `--color-accent-soft` | `#EEE4CF` | 激活底色 |
| `--color-focus` | `#356A8A` | 键盘焦点 |
| `--color-danger` | `#9E3F3A` | 禁止出现、错误 |
| `--color-danger-soft` | `#F4E1DE` | 禁止区域底色 |
| `--color-warning` | `#9A6826` | 尽量避开、警告 |

| `--color-success` | `#3E725B` | 成功和已满足 |

所有组合需达到 WCAG AA 正文对比度。不能只用颜色传达必选、禁止、缺失、激活或不可添加状态。

## 3. 字体与排版

```css
font-family: Inter, "Noto Sans SC", "Microsoft YaHei UI", "PingFang SC", system-ui, sans-serif;
```

不依赖 Google Fonts 或运行时网络字体。正文 14–16 px，行高 1.5；面板标题 16–18 px、600 weight；页面标题不超过 24 px。数字和 hash 可使用系统等宽字体，但技能名与正文不用等宽字体。

## 4. 空间、形状和层级

- 间距使用 4 px 基线：4、8、12、16、24、32。
- 常规控件圆角 6 px，面板圆角 8 px；胶囊仅用于短技能 chip。
- 主面板 1 px 边框；激活目标 2 px 雾金边框。
- 阴影只用于弹层：`0 8px 24px rgba(28, 40, 51, 0.14)`；普通面板不悬浮。
- 动画 120–180 ms，只改变颜色、透明度或阴影；遵循 `prefers-reduced-motion`。

## 5. 核心组件

### 目标分区

- 默认：象牙白表面、1 px 中性边框。
- 激活：2 px `--color-accent`、`--color-accent-soft` 底色，并显示“正在添加到这里”。
- 点击标题/空白区域切换激活；点击已有技能只删除一个 occurrence。
- “禁止出现”与“尽量避开”使用各自语义边框和图标，但不抢夺当前激活状态的视觉层级。

### 技能项

- 自动换行的紧凑 chip，最小高度 32 px，文字不截断到无法区分。
- 可添加：明确 hover 和 pointer；不可添加：降低对比度、锁图标、`not-allowed`，但仍可聚焦查看原因。
- 重复 occurrence 默认逐项显示；只有相邻同值的连续项可以压缩成 `×N`，且内部仍保留 entry ID 与顺序。禁止把 `[A,B,A]` 全局聚合；删除一次固定删除一个 occurrence。

### 帮助图标与气泡

- 使用 Lucide `CircleHelp`，视觉 16–18 px，交互范围至少 32 px。
- hover 与 focus 均触发；约 250 ms 打开，`Esc` 关闭，最大宽度 320 px。
- 图标有 `aria-label="了解此区域"`；气泡与触发器建立无障碍关联。

### 按钮与输入

- 主按钮使用夜蓝实底，不使用橙色营销 CTA。
- 次按钮使用透明底和中性边框；危险按钮仅在确有删除风险时使用红色。
- 焦点环统一为 2 px `--color-focus` 加 2 px offset，不移除 outline 后留空。
- 搜索、选择和开关都有可见标签或可访问名称，错误文本紧邻控件。

## 6. 桌面布局

- 顶部提供紧凑的方案名称、保存状态和方案切换，不制作网站导航栏。
- 目标编辑页左右视觉分区但不显示“技能池/目标技能”总标题；左侧搜索分类，右侧子目标纵向排列。
- 结果区优先展示因子的主/副词条顺序、数量聚合、目标缺失和排序原因；不使用角色立绘作为主要信息载体。
- 窄窗口允许目标区纵向滚动或切换为顺序布局，核心操作不得依赖横向滚动。

## 7. 文案规则

使用“必须满足、基础能力主词条、可选目标、基础词条替代顺序、禁止出现、尽量避开”等直接名词。避免“硬屏蔽、软屏蔽、token、canonical、solver”等内部术语出现在普通界面。错误说明给出原因和下一步，不使用拟人化或夸张语气。

## 8. 交付检查

- [ ] 没有 emoji 充当图标，图标来自同一套 Lucide。
- [ ] 所有点击区域有 hover、pointer 和可见键盘焦点。
- [ ] tooltip 支持 hover、focus、Escape 和屏幕阅读器。
- [ ] 不可添加项仍可聚焦并能说明原因。
- [ ] 颜色对比满足 WCAG AA，语义不只依赖颜色。
- [ ] 无外部字体、远程图片、运行时代码和非必要网络请求。
- [ ] 目标交互符合 `docs/architecture/block-pool-ui.md`。
- [ ] 1024、1280、1440、1920 px 宽度和 Windows 125%/150% 缩放完成视觉检查。
- [ ] 截图对比未与参考站形成近似布局、配色或素材印象。
