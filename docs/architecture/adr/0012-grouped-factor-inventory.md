# ADR 0012：按组合和数量管理因子库存

- 状态：Accepted
- 日期：2026-07-24

## 背景

同一主词条、副词条和因子等级的多枚因子在配装业务中可以互换。库存位置来自存档结构，玩家不需要看到，也不应决定某套配装是否仍然有效。

旧模型使用具体库存实例作为确认配装的占用身份。存档中的位置变化会让内容相同的因子被误判为缺失，替换列表也会重复显示大量相同选项。

## 决定

存档读取层继续保留 `RawSigil` 的原始身份，只用于只读解析校验、诊断和向求解器展开候选。业务层使用版本化库存组：

```text
group-v1:<有序主词条>:<副词条>:lv<因子等级>
```

每组记录总数、已确认数量和当前可用数量。主副顺序不同或因子等级不同，必须属于不同组。

确认配装保存 `groupKey + count` 的多重计数。改用另一套配装时，只扣除其他方案的占用，再原子替换当前方案的旧计数。任何占用数量变化都会保守地使相关分析缓存失效。

界面不显示库存位置。相同库存候选只显示一项，并使用“可用 N”说明剩余数量。

## 重新读取存档

重新读取后按库存组重新统计数量，不按位置查找旧实例。如果全部确认配装在某组的需要量超过现有数量，所有使用该短缺组的配装都标为需要检查；工具不自动解除、替换或选择优先保留的角色。

旧工作区迁移时保留目标方案。已确认记录优先从保存的结果快照推导组和数量；旧分析与手动结果缓存失效后重新计算。无法推导的遗留确认记录继续按旧实例检查，直到用户取消或重新确认。

## 游戏内装备状态

游戏内装备数量作为组内信息展示，暂不改变既有求解语义，避免与应用内确认配装重复扣减。以后若将游戏内装备视为不可用库存，需要单独制定角色识别和扣减策略。

## 结果

- 库存列表和替换候选更短。
- 同内容因子的位置变化不会破坏确认记录。
- 重复因子的占用必须使用计数，不能使用集合。
- 求解器仍可接收展开后的临时候选，但持久化占用不依赖临时实体顺序。
