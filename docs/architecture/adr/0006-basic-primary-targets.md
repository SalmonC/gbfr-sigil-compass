# ADR 0006：将基础属性主词条建模为复合软目标

- 状态：Accepted
- 日期：2026-07-22

## 背景

Master Trait 按因子主词条类别计数，使“技能出现”与“技能处于主词条”成为不同约束。用户还需要指定 X 个基础属性目标，并在库存缺少精确目标时按有序白名单替代。

## 决策

目标总容量仍为 24，划分为普通目标与基础属性主词条目标。后者由有序技能、主词条位置和可选有序替代池组成；它是仅次于必选技能的软目标，不过滤未满 X 的方案。

求解器使用统一的一对一 token 分配连接普通目标、基础属性目标与已选因子的主副词条。一个主词条 token 不能同时满足普通目标和基础属性目标；必选普通目标先获得容量。基础属性内部比较顺序为已填充数量、精确覆盖向量、替代池使用向量。基础属性类别与主词条合法性由版本化 Catalog 提供，UI 和求解器不按名称判断。

## 后果

- `BuildRequest` 分离普通目标和 `BasicPrimaryTargetPolicy`。
- `BuildResult` 为每个基础属性目标返回精确、替代或缺失的解释。
- 配置编码必须保存替代池顺序、两个开关和 Catalog 版本。
- 主词条目标数量不能超过可装备槽位数。
- 需要用二分图匹配边界、重复目标和动态替代池顺序做领域测试。

完整比较器已由后续双层屏蔽需求提升为 [`GBFR-RANK-2`](../ranking-specification.md)。
