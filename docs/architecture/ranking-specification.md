# 配装匹配与排序规格（GBFR-RANK-4）

## 1. 适用范围

本文件冻结当前求解器的业务语义。求解结果、缓存和分享字符串记录版本 `GBFR-RANK-4`。本版沿用 RANK-3 的三类主词条和双层屏蔽规则，并调整结果去重：目标外副词条的具体内容不再单独生成结果，但其所在主词条位置仍参与方案身份。改变排序层级、目标匹配或方案等价关系时必须再次提升版本。

## 2. 目标编辑模型

目标方案包含八个选择域：

- `mandatory`：必须满足，有序、可重复；
- `basicPrimary`：基础属性主词条，有序、可重复；
- `attackPrimary`：攻击类主词条，有序、可重复；
- `defensePrimary`：防御类主词条，有序、可重复；
- `optional`：可选目标，有序、可重复；
- `basicSubstitutionOrder`：基础属性替代池，有序、不可重复；
- `forbidden`：不能出现，无序、不可重复；
- `avoid`：尽量避开，无序、不可重复。

必须满足、三类主词条和可选目标合计最多 24 项。三类主词条合计最多 12 项，因为每枚因子只有一个主词条。界面只在“可选目标”显示动态容量：

```text
24 - mandatory.Count - basicPrimary.Count - attackPrimary.Count - defensePrimary.Count
```

各主词条域只能添加 Catalog 中相应类别且允许作为主词条的技能。`mandatory` 与 `optional` 允许共享 Skill ID；其余跨域组合互斥。

## 3. 技能 token 分配

每枚因子的主、副词条各提供一次技能出现，同一次出现只能匹配一个目标。统一分配顺序为：

1. 必须满足；
2. 已开启“优先满足”的三类主词条；
3. 可选目标，以及未开启优先满足的主词条目标。

必须满足优先消费副词条；只有副词条不足时才消费同技能的主词条，从而给第二层保留尽可能多的主词条。三类主词条共同使用一个主词条池，一枚因子最多填一个主词条位置。

## 4. 三类主词条

每个主词条域有独立的“优先满足”开关。

- 开启后，该域进入第二优先级；未全部满足的方案仍可输出，但要标明缺口。
- 关闭后，主词条位置约束不参与求解，其中的技能按 `optional → basicPrimary → attackPrimary → defensePrimary` 的顺序作为普通可选目标比较。

所有已开启域先合计比较完成数量，再按 `basicPrimary → attackPrimary → defensePrimary` 以及各域内部顺序比较精确覆盖向量。不能为了满足较早位置而减少总完成数。

只有基础属性支持替代。精确基础主词条不足、且已开启替代时，按 `basicSubstitutionOrder` 从前往后补足基础属性缺口；攻击和防御缺口不能用其他技能替代。替代池先比较第 1 项的实际使用数，再比较第 2 项，依次类推。

## 5. 可行性条件

必须满足按多重集计数。任一选中因子的主词条或副词条命中 `forbidden`，整个候选淘汰。两者同属硬条件：

- 必须满足缺少一次，不输出；
- 不能出现的技能命中一次，不输出。

`avoid` 不删除候选，按最终实际出现次数计数。`X&X` 在 X 属于尽量避开时记两次。

## 6. 全局比较器

候选方案按以下顺序逐级比较，上一层完全相同才进入下一层：

1. 必须满足全部达成，且不能出现的技能为 0；否则淘汰。
2. 已开启优先满足的主词条完成总数降序。
3. 按基础、攻击、防御目标顺序形成的精确覆盖向量降序。
4. 按基础替代池顺序形成的实际使用数量向量降序。
5. 可选目标完成总数降序。
6. 尽量避开的技能出现次数升序。
7. 按可选目标顺序形成的覆盖向量降序。
8. 使用因子数升序。
9. 所选实体因子的 `sigilLevel` 总和降序。
10. 基于 `runSeed` 和逻辑因子类型生成的两组稳定线性键。
11. canonical signature 升序。

主词条完成度高于可选目标，因此允许牺牲可选目标换取更多已开启的主词条目标。可选目标完成数高于尽量避开，因此多完成一个目标的方案不会仅因软屏蔽而被更少目标的方案超过。

## 7. 方案身份与实例

先把每枚因子规范化为：

```text
(primarySkillId, secondaryRole)
secondaryRole = 命中任一目标时保留 secondarySkillId，否则使用通配符 *
```

逻辑方案身份是上述规范化有序词条对的数量多重集合，并同时包含目标覆盖结果。`A&B` 与 `B&A` 是不同方案。两个方案只有目标外副词条内容不同，且通配符落在相同主词条上时视为同一方案；如果目标外词条落在不同主词条上，仍是不同方案，因为手动替换时可选的库存因子不同。

等级、实例 ID、库存位置和装备角色不进入方案等价判断。同一逻辑方案存在多组实现时，按既有比较器保留更优的一组。库存占用按“有序主词条 + 副词条 + 因子等级 + 数量”记录，库存位置不进入业务身份。

## 8. 输出解释

每套结果至少显示：

- 可选目标完成数量；
- 已开启主词条的 `完成数/目标数`；
- 基础属性替代和缺失；
- 尽量避开的技能及出现位置；
- 因子数、每枚因子等级与库存组；
- 按技能汇总的实际等级，例如两条 15 级追击显示为“追击 Lv 30”；
- 与上一名首次产生差异的排序层级。

目标外词条使用弱化样式，但仍保留技能名、图标和等级。特殊收集型技能按 Catalog 规则显示实际说明，不把因子等级伪装成技能等级。主词条未满、发生基础替代或带有尽量避开的技能时，在结果标签和结果顶部同时提示。不能出现的技能不会出现在结果中；无解时说明必须满足或禁用条件无法同时成立。

用户可在结果中固定一个词条并从真实、未占用库存中替换另一个词条，也可替换、删除或添加整枚因子。相同主副词条和等级的候选按数量合并。手动调整版挂在原排名结果下，不重新参与 Top-10 排序；每次编辑后重新检查目标、屏蔽和技能等级汇总。确认配装时占用调整后所需的库存组数量。

## 9. 版本与迁移

- schema 2 方案迁移到 schema 3 时，新增空的攻击、防御主词条域和关闭的优先开关。
- `GBFR-RANK-2`、`GBFR-RANK-3` 分享串可读取；重新导出使用 `GBFR-RANK-4`。
- 旧版分析缓存不能冒充 RANK-4 结果，加载时直接删除；目标方案本身继续保留。
- v3 已确认配装优先从结果快照迁移为库存组计数；旧分析缓存失效后重新计算。

## 10. 最小验收用例

1. 必须满足缺少一次时不输出任何方案。
2. 主、副任一词条命中不能出现的技能时，该因子不进入结果。
3. 三类主词条共 13 项或所有目标共 25 项时拒绝保存和分析。
4. 三类主词条总完成数更多的方案高于可选目标完成更多但主词条更少的方案。
5. 总完成数相同时，较早的基础、攻击、防御精确目标依次优先。
6. 基础属性替代只补基础缺口，不能补攻击或防御缺口。
7. 一枚因子的主词条不能同时满足两个主词条目标。
8. 关闭某类优先满足后，其技能作为普通可选目标参与比较。
9. 尽量避开次数只在可选目标完成数相同后生效，`X&X` 记两次。
10. 同内容、不同等级的实例属于同一逻辑方案，并选择等级评分较高的实现。
11. `A&B` 与 `B&A` 输出为两个方案。
12. 只有目标外副词条不同且通配符所在主词条相同的结果合并；通配符位置不同的结果不合并。
13. 相同库存、目标、Catalog、RANK-4 和 seed 返回同序前十名。
