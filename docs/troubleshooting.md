# 故障排查

## Windows 阻止运行

当前版本没有商业代码签名，可能触发 SmartScreen。请只从本仓库 Release 下载，并核对 `SHA256SUMS.txt`。来源不明或哈希不一致的文件不要运行。

## 双击后没有窗口

1. 确认 ZIP 已完整解压；
2. 不要只复制 `Sigil-Compass.exe`，它需要同目录中的其他文件；
3. 尝试移动到纯英文、权限正常的目录；
4. 暂时检查安全软件的隔离记录；
5. 提交 Issue 时附上 Windows 版本和解压后的目录结构截图。

## 找不到存档

按 `Win + R`，打开：

```text
%LOCALAPPDATA%\GBFR\Saved\SaveGames\
```

通常选择 `SaveData1.dat`。如果使用了不同 Steam 用户或移动过系统目录，请按文件修改时间确认，不要选择应用自己生成的缓存文件。

## 读取失败或因子数量不对

- 确认游戏已经正常保存并完全结束写入；
- 重新选择修改时间最新的 `SaveData*.dat`；
- 不要上传公开存档来求助；
- 如果游戏刚更新，可能是存档格式或技能目录尚未适配。

应用会拒绝校验不通过或结构异常的文件，不会尝试修复存档。

## 游戏内库存变了，工具里没变

这是正常行为。应用启动时不会自动重读存档，请手动点击“读取存档”更新快照。

## 提示计算过于复杂

当前精确求解器有 30 秒和状态容量保护。可尝试：

- 减少可选目标；
- 删除不重要的重复目标；
- 增加“不能出现”的技能，缩小候选范围；
- 先保存当前方案，再分几组目标排查是哪一项造成组合爆炸。

触发上限后不会缓存或显示不完整结果。

## 某套方案少了已确认因子

重新读取存档后，如果已占用的具体实例被分解、变更或无法识别，工具会将其标为缺失。取消旧配装后重新计算即可。它不会自动拿同词条因子顶替，以免误占其他角色的装备。

## 清空本地缓存

先关闭应用，再删除：

```text
%APPDATA%\Sigil Compass\
```

这会清除库存快照、本地方案、计算缓存和确认配装，但不会删除或修改游戏存档。不同构建下目录名称可能略有差异，可按修改时间确认。

## 如何提交有效反馈

Issue 中请提供：

- 应用版本和 Windows 版本；
- 问题发生前的操作步骤；
- 完整错误提示；
- 已遮挡个人信息的截图；
- 是否能稳定复现。

请勿上传 `SaveData*.dat`。确实需要分析敏感样本时，先通过 Security 页面私下联系。
