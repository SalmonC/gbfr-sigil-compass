# 游戏技能图标使用边界审计

日期：2026-07-23
结论性质：工程发行建议，不构成法律意见。

## 结论

从社区项目或 Wiki 下载图标，不能自动取得游戏图标的复制和再分发权。若图片与游戏内图标完全一致，权利判断仍回到原始美术素材，而不是下载页面。

Cygames 的 Relink 官方站声明，站内内容权利归 Cygames 或第三方所有；除法律允许的私人使用外，未经事先同意不得复制、再发布、修改或公开传输。该页面没有给本工具可直接依赖的软件素材许可：[Cygames — About This Website](https://relink.granbluefantasy.jp/en/about/)。

Granblue Fantasy Wiki 自己也把两层权利分开：页面贡献内容采用 CC BY-NC-SA，但 Granblue Fantasy 素材的商标与版权仍属于 Cygames 或其许可方。其文件页明确显示这一保留条款：[Granblue Fantasy Wiki — Relink main visual file page](https://gbf.wiki/File%3ARelink_main_visual.jpg)。Creative Commons 的官方 FAQ 同样说明，作品中的第三方材料不当然落入页面的 CC 许可，使用者仍需确认第三方材料本身的许可或法定例外：[Creative Commons FAQ](https://creativecommons.org/faq/)。

因此：

1. “社区项目不是原创”不能作为可再分发的许可证；它至多说明文件的获取路径。
2. 上游仓库没有许可证时，既不能假定获得其代码/整理成果许可，也不能由它代替 Cygames 授权游戏原图。
3. 本地、私人调试与向公众打包分发是两个风险不同的场景。官方站明确提到私人使用例外，但具体范围取决于适用法；不能把它扩张解释为公开发布许可。
4. 免费、开源、注明出处或不盈利，都不会单独产生复制许可。它们可能影响实际风险评估，但不是授权本身。

## 对本项目的可执行方案

按风险从低到高：

1. 公开包只带项目原创的类别占位图；允许用户在本地选择自行取得的图标包，图片不进入安装包和仓库。
2. 在用户已安装游戏的前提下，由本地工具从其合法副本提取图标，生成仅供该用户使用的缓存；需另行审查游戏条款和提取技术边界。
3. 向 Cygames 取得对非商业配装工具分发指定图标的书面许可，并按许可加入署名、用途、地域、期限和撤回机制。
4. 未取得许可前直接把原始游戏图标打入公开 Windows/macOS 包，不作为当前推荐发布方案。

当前仓库中的游戏图标只保留为本地验证素材。正式公开发行闸门仍是“取得明确许可”或“从公开包移除并采用用户本地导入/提取”。
