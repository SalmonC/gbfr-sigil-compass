# 截图基准测试方案

## 用途

本文件冻结用户提供截图中的前 16 个技能及新增规则，作为目标编辑器、配置存储、分享字符串和求解器的共同验收夹具。正式自动化夹具必须使用 Catalog 的稳定 Skill ID/hash；本文件中的中文是输入证据，不得仅凭显示名猜测身份。

## 方案名称

`欧羽尼高手-截图前16项`

分享字符串验收时必须编码并还原该名称；不得包含存档路径、库存、玩家名或 Steam ID。

## 必须满足

按顺序、按 occurrence 记录，共 8 项：

1. 天星之煌（Celestial Lumen）
2. 天星之界（Celestial Terra）
3. 天星止息（截图原文；Catalog 候选 Celestial Ventus，显示译名待 2.0.2 简中表核验）
4. 天星之雪（Celestial Aqua）
5. 浪迹天涯（Fatebreaker）
6. 伤害上限（Damage Cap）
7. 伤害上限（Damage Cap）
8. 伤害上限（Damage Cap）

“必须满足”标题不显示 `8 / ?` 或其他容量徽标，只显示帮助问号。

伤害上限还会在下方“可选目标”再出现一次；这是必须满足/可选目标共享 Skill ID 的强制回归用例，不能被跨区互斥规则拒绝。

## 可选目标

按顺序、按 occurrence 记录，共 8 项：

1. 伤害上限（Damage Cap）
2. 狂战士（Berserker Echo，trait hash `0xEE85CD1F`）
3. 斯巴达（Spartan Echo）
4. 追击（Supplementary DMG）
5. 追击（Supplementary DMG）
6. 追击（Supplementary DMG）
7. 躲避仇火（截图原文；Catalog 候选 Dodge Payback，中文待核验）
8. 属转（截图原文；Catalog 候选 War Elemental / 属性克制转换，中文待核验）

基础能力主词条有 5 项、必须满足有 8 项，因此可选目标上限为 `24 - 5 - 8 = 11`。界面应显示 `8 / 11`，问号气泡中的实时剩余数为 3。

## 基础能力主词条

- 开启“优先保证基础主词条”（对应用户所说的主属性必选/强制选项）。
- 昏厥 ×3。
- 体力 ×2。
- “允许基础词条替代”默认关闭；替代顺序为空。

该区域不显示 `5 / 5` 容量徽标，但保留帮助问号。若以后专门测试替代，需要在独立用例中开启开关并指定有序替代池，不能擅自改变本夹具。

## 禁止出现

- 刀上舞（Glass Cannon）。
- 穷寇心（Berserker，trait hash `0x70395731`）。

`Berserker` 与 `Berserker Echo` 是两个不同稳定技能；本夹具故意同时把前者放入“禁止出现”、后者放入“可选目标”，用于阻止按中文近似名合并。

## 尽量避开

选择三个当前目录证据可确认的 Defense 技能：

- Aegis。
- Improved Guard。
- Steel Nerves。

中文显示名必须由发布 Catalog 提供，不在夹具中手写临时译名。这三个技能是无序集合；改变点击顺序不得改变保存 hash 或结果排序。

## 编辑器验收

1. 依次激活六个子目标并从左侧技能池添加，结果与上文一致。
2. 必须满足、基础能力主词条、可选目标允许重复；另三个区域不允许重复。
3. 点击右侧一个“追击”只删除一次 occurrence，不取消当前激活区域。
4. 技能被一个区域占用后，在与之互斥的其他区域下保持可见但不可添加，并说明占用位置；必须满足与可选目标之间是明确例外。
5. 基础能力主词条激活时，非 Basic Stats 技能不可添加并说明类别限制。
6. 只有可选目标显示 `8 / 11`；六个区域都显示可悬停和可聚焦的帮助问号。
7. 保存、关闭、重新加载和分享字符串导入后，三个有序目标的顺序/重复次数、两个无序集合和方案名称完全一致。

## 求解验收

- 任一结果都满足 8 个“必须满足” occurrence，且不含刀上舞或穷寇心。
- 开启优先保证基础主词条后，先比较五个基础能力主词条的满足数量和质量，再比较可选目标；未满五个时仍输出并标出缺口。
- 可选目标满足数相同时，带有更少“尽量避开”出现次数的方案优先；它不能越过满足更多可选目标的方案。
- 逻辑方案忽略实体 ID 和等级，但保留主副顺序；同一逻辑方案选择 `sigilLevel` 总和最高的实体实现。
- 最多展示前 10 个逻辑方案。
