# 更新记录

## 开发中

- 相同主、副词条和因子等级的库存合并显示，替换候选使用“可用 N”展示数量。
- 确认配装改为按词条组合、等级和数量占用；库存位置不再显示或参与业务占用。
- 持有因子页面改为聚合库存，支持查看持有、可用、已确认和游戏内装备数量。
- 技能汇总按基础属性、攻击、防御、辅助/特殊、角色专属排列。
- README 加入读取存档、目标编辑和计算设置截图，并拆分单词条修改与整枚因子替换说明。

## v0.2.3

- 结果中不属于任何目标的附带词条改为灰色显示。
- 主、副词条支持从真实未占用库存中替换，也可以添加、删除或更换整枚因子；调整版随原方案缓存。
- 因子列表下方按技能汇总等级，不再显示因子等级总和。
- 调整结果去重规则：目标外副词条只有位于相同主词条位置时才会合并。
- 修复部分窗口宽度下顶部库存摘要逐字换行的问题。

## v0.2.2

- 项目原创代码和文档改用 MIT License；游戏内容及第三方素材不在该授权范围内。
- 优化精确 Top-10 求解器：等价贡献合并、紧凑状态和延迟分配显著降低复杂库存的时间与内存。
- 新增可调的快速计算内存，默认 512 MB，可设为 128–2048 MB；达到切换线后自动改用低内存精确求解，不再报组合容量不足。
- 内置 HiGHS WebAssembly 精确兜底，不写磁盘缓存，复杂配置仍保持完整排序。
- 修复方案操作菜单点击外部不关闭，以及菜单与帮助气泡的层级冲突。

## v0.2.1

- 单次计算时间上限可在界面设置为 5–600 秒，默认 30 秒并记住上次设置。
- 分开提示“计算超时”和“组合状态超过安全容量”。
- 顶部增加 GitHub 项目入口。

## v0.2.0

- 在读取按钮旁补充默认存档目录和文件选择说明。
- 精确求解时间预算提高到 30 秒，外层 Worker 在 35 秒回收。
- 放宽求解状态容量，仍然拒绝返回不完整结果。
- 修复 Windows 宽屏下技能池内容高度与外框不一致的问题。
- 增加相应回归检查。

## v0.1.0

- 首个可用版本。
- 支持只读存档导入、目标编辑、精确 Top-10 配装、方案缓存与分享。
- 支持库存查看、确认配装和多角色具体因子占用。
